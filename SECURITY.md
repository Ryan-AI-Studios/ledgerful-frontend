# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the Ledgerful dashboard, please report it responsibly:

- **Email:** security@ledgerful.dev
- **GitHub Private Vulnerability Reporting:** Use the [GitHub security advisory feature](https://github.com/Ryan-AI-Studios/ledgerful-frontend/security/advisories/new)

Please **do not** open a public GitHub issue for security reports.

## Response Timeline

| Stage | Target |
|-------|--------|
| Acknowledgment | Within 3 days |
| Initial assessment | Within 7 days |
| Fix or mitigation | Within 30 days |
| Public disclosure | After fix is released, coordinated with reporter |

## Scope

This policy covers the Ledgerful frontend dashboard (`ledgerful-frontend`). For the engine/daemon, see the [engine SECURITY.md](https://github.com/Ryan-AI-Studios/Ledgerful/blob/main/SECURITY.md).

## Safe Harbor

Good-faith security research is welcomed. We will not pursue legal action against reporters who:

- Avoid accessing or modifying data that is not their own.
- Do not degrade or disrupt services.
- Report findings promptly and do not publicly disclose them before a fix is released.
- Do not use social engineering or physical attacks.

## Supported Versions

Only the latest release line receives security fixes.
